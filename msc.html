BlitzBox

BlitzBox is a cross-platform and cross-play multiplayer arena shooter built with a team of 8 people. The game is built around a gravity mechanic allowing for each player to rotate the direction of their own gravity at any moment changing their position in the cube level. In this project I was the head of AI and single player and these are my contributions:

  • Generating and exporting navmesh data from Unity for all sides of the cube
  • Importing navmeshes into the project and modifying them to work with our framework
  • Creating a pathfinding algorithm that works on all sides of the cube level
  • Creating the games AI and handling their behaviour through state machines to allow for:
    • The AI to switch between default behaviour and attacking behaviour when the player is nearby
    • The AI to react to and pathfind to the player
    • The AI to attack the player when appropriate
    • The AI to handle different states and make decisions when appropriate considering states such as the reload timer
  • Designed and created the single player game mode including the following:
    • A wave based system where the player must defeat a certain number of AI to continue
    • The AI progressively gets harder as the game continues by getting more aggressive and more dangerous
    • Creating a scoring system and handling game ending to be functional with the game's wider menu loop

OpenGL Graphics Project

This project is built to demonstrate advanced graphics techniques created with OpenGL and shaders. It contains multiple scenes which can be switched between that each demonstrate different effects. These are some of the effects and technical details created for the project:

  • Generated landscapes with multiple blended textures
  • Standard lighting techniques using ambient, diffuse, and specular components
  • Skybox textures used in combination with environment mapping to show water reflections
  • Loading and animating meshes with the appropriate textures
  • Deferred rendering and advanced buffers to allow for:
    • A post-processing blur effect
    • Dynamically rendering up to 256 moving light sources
    • A split-screen effect to show both scenes simultaneously

Game of Life C++

This university project demonstrates the use of advanced C++ features to create an optimized simulation for Conway's Game of Life. It can accurately run simulations on varying board sizes and search for specific patterns, including spaceships. This codebase has been built from scratch using object-orientation and utilizes many C++ features including operator overloading, templates, threading, and interfaces.
